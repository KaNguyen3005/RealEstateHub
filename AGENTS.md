## Coding Rules

- Implement one task at a time.
- Do not generate the whole project at once.
- Do not change the tech stack.
- Do not hardcode secrets.
- Follow Router → Controller → Service → Model.
- Use standard API response format.
- Add validation and error handling.
- Keep UI responsive.
- Update .env.example when adding env variables.
- Stop and ask if requirements are ambiguous.

## After Each Task

- List changed files.
- Explain how to run/test.
- Mention unfinished items.
